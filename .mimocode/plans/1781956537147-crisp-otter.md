# AI Job Interview Simulator вЂ” Full Plan

## Context

Kaggle Capstone project. Greenfield вЂ” not a single code file exists. Need to build a full AI Interview Simulator with 5 agents, deploy to Google Cloud Run.

**Deadline:** July 6, 2026
**Stack:** Google ADK (TS) + DeepSeek via OpenRouter + Fastify + Next.js + Redis + Cloud Run

---

## Architecture Decisions

### LLM: DeepSeek via OpenRouter
- TypeScript ADK doesn't have a built-in LiteLLM connector (unlike Python)
- Decision: use **Google GenAI SDK** with custom `baseUrl` pointing to OpenRouter
- OpenRouter is compatible with OpenAI API format
- Model: `deepseek/deepseek-chat` via OpenRouter

### ADK: TypeScript (`@google/adk`)
- Official package `@google/adk`
- `LlmAgent` + `FunctionTool` for each of the 5 agents
- `SequentialWorkflowAgent` for orchestration pipeline

### Storage: Redis (ioredis)
- Simpler for dev + Cloud Run (stateless)

## Phase 1: Minimal Chat with DeepSeek (Day 1-2)

**Goal:** Deploy a working chat bot on Cloud Run. No agents вЂ” just a direct DeepSeek call via OpenRouter.

**File:** `src/config.ts`

Create a wrapper over the OpenRouter API. OpenRouter uses OpenAI-compatible API:
- Base URL: `https://openrouter.ai/api/v1`
- Headers: `Authorization: Bearer $OPENROUTER_API_KEY`
- Model: `deepseek/deepseek-chat`

### Step 1.2: Minimal ADK Agent
**File:** `src/agents/root.ts`

```typescript

export const rootAgent = new LlmAgent({
  name: 'interview_bot',
  model: 'gemini-flash-latest', // temporarily use Gemini for dev
  instruction: 'You are a helpful interview assistant.',
});
```

> **Important:** TypeScript ADK works with Gemini by default. For DeepSeek you need:
> 1. Either use `GEMINI_API_KEY` for dev/testing
> 2. Or configure OpenRouter в†’ ADK proxy via httpOptions
> 3. Or write a custom ModelAdapter

**Recommendation for Phase 1:** Use free Gemini API for rapid prototyping, connect DeepSeek in Phase 2.

## Phase 2: 5 ADK Agents (Day 3-5)

### Step 2.1: Type Definitions
**File:** `src/types/index.ts`

```typescript
interface JobProfile {
  role: string;
  level: 'junior' | 'middle' | 'senior';
  skills: string[];
  domain: string;
  keywords: string[];
}

interface Question {
  id: string;
  text: string;
  topic: string;
  difficulty: number;
}

interface Evaluation {
  score: number;
  strengths: string[];
  weaknesses: string[];
  recommendation: string;
}

interface Session {
  id: string;
  jobProfile: JobProfile;
  history: Array<{ question: string; answer: string; evaluation: Evaluation }>;
  weakTopics: string[];
  currentDifficulty: number;
}
```

### Step 2.2: FunctionTools
**Files:** `src/tools/*.ts`

Each tool is a `FunctionTool` from `@google/adk`:

1. **`parseJob.ts`** вЂ” parses job description text в†’ `JobProfile`
   - LLM call with structured output (JSON schema)
   - Writes result to Redis

2. **`generateQuestion.ts`** вЂ” generates question based on `JobProfile` + `weakTopics`
   - Considers current difficulty level
   - Focuses on weak topics

3. **`evaluateAnswer.ts`** вЂ” evaluates answer в†’ `Evaluation`
   - Structured output: score, strengths, weaknesses

4. **`updateMemory.ts`** вЂ” updates `weakTopics` in session
   - Adds/removes weak topics

5. **`fetchWeakTopics.ts`** вЂ” fetches current weak topics

### Step 2.3: Agents
**Files:** `src/agents/*.ts`

1. **JobParserAgent** вЂ” uses `parseJob` tool
2. **InterviewerAgent** вЂ” uses `generateQuestion` tool
3. **EvaluatorAgent** вЂ” uses `evaluateAnswer` tool
4. **CoachAgent** вЂ” explains correct answer, gives tips
5. **MemoryAgent** вЂ” uses `updateMemory` + `fetchWeakTopics`

### Step 2.4: Orchestration (Sequential Workflow)
**File:** `src/agents/root.ts`

```
Pipeline:
1. User enters job description в†’ JobParser в†’ creates JobProfile
2. Interviewer в†’ generates question
3. User answers в†’ Evaluator в†’ evaluates
4. Coach в†’ explains
5. Memory в†’ updates weak topics
6. Interviewer в†’ next question (loop)
```

Use `SequentialWorkflowAgent` or manual orchestration via Fastify routes.

### Step 2.5: API Routes
**Files:** `src/routes/*.ts`

- `POST /job/parse` вЂ” parses job description, creates session
- `POST /interview/start` вЂ” starts interview
- `POST /interview/answer` вЂ” accepts answer, returns next question + feedback
- `GET /session/:id` вЂ” gets session data

### Step 2.6: Redis Service

