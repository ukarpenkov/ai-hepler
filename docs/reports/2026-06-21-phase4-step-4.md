# Phase 4 — Шаг 4: Tool Access Control

**Статус:** Выполнено  
**Дата:** 2026-06-21

## Созданные файлы

- `src/security/toolAccess.ts` — модуль контроля доступа к tools
- `src/security/toolAccess.test.ts` — unit тесты

## Реализовано

1. Класс `ToolAccessGuard`:
   - `registerTool(name, accessLevel)` — регистрация tool с уровнем доступа
   - `checkAccess(toolName, callerContext)` — проверка доступа
   - `hasTool(name)` — проверка существования
2. Матрица доступа:
   - `system` tools (parseJob, generateQuestion, evaluateAnswer) → доступны только из "agent"
   - `memory` tools (updateMemory, fetchWeakTopics) → доступны из "agent" и "route"
3. Singleton `defaultGuard` с зарегистрированными 5 tools
4. Интеграция в agents: проверка `defaultGuard.checkAccess()` перед вызовом tools

## Интеграция в agents

- `job-parser.agent.ts` — проверка перед `parseJobDescriptionTool`
- `interviewer.agent.ts` — проверка перед `generateQuestionTool`
- `evaluator.agent.ts` — проверка перед `evaluateAnswerTool`
- `memory.agent.ts` — проверка перед `updateMemoryTool` и `fetchWeakTopicsTool`
