## Implementation

1. In-memory rate limiter based on `Map<string, RateLimitEntry>`
2. Configuration: `windowMs` (default 60000) + `maxRequests` (default 30)
3. Fastify plugin via `fastify-plugin` for bypass encapsulation
4. Rate limit key — IP address (with `X-Forwarded-For` support)
5. When limit exceeded — 429 with `{ error, retryAfter }`
6. Headers `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
7. Automatic cleanup of expired entries via `setInterval`
