## Architecture

### Agent Pattern
All agents follow the same structure:
1. Extend `LlmAgent` from `@google/adk`
2. Read config from `process.env.LLM_MODEL || "deepseek/deepseek-chat"`
3. Attach tools (where applicable) via `tools` array
4. Set `outputKey` for session state propagation

### Pipeline Data Flow
```
User Answer
    ‚Üì
EvaluatorAgent ‚Üí evaluation
    ‚Üì
CoachAgent ‚Üí coachFeedback
    ‚Üì
MemoryAgent ‚Üí memoryUpdate (weakSkills)
    ‚Üì
InterviewerAgent ‚Üí currentQuestion
```

### Runner Architecture
- `jobParserRunner`: standalone runner for job parsing (Step 16-17 will integrate)
- `interviewRunner`: runs the full interview pipeline
- Shared `InMemorySessionService` for session management
