# Phase 4 — Шаг 2: Rate Limiter middleware

**Статус:** Выполнено  
**Дата:** 2026-06-21

## Созданные файлы

- `src/security/rateLimiter.ts` — in-memory rate limiter middleware
- `src/security/rateLimiter.test.ts` — unit тесты

## Реализовано

1. In-memory rate limiter на основе `Map<string, RateLimitEntry>`
2. Конфигурация: `windowMs` (дефолт 60000) + `maxRequests` (дефолт 30)
3. Fastify plugin через `fastify-plugin` для bypass encapsulation
4. Ключ rate limit — IP адрес (с поддержкой `X-Forwarded-For`)
5. При превышении лимита — 429 с `{ error, retryAfter }`
6. Заголовки `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
7. Автоматическая очистка устаревших записей через `setInterval`

## Экспорт

- `createRateLimiter` — Fastify plugin (опциональные `windowMs`, `maxRequests`)
- `rateLimitStore` — Map для тестов
