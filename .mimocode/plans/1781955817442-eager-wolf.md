## Что НЕ хватает для начала

### Критично (нужно ДО начала кода)

1. **`package.json`** — нет вообще, невозможно стартовать
2. **`tsconfig.json`** — TypeScript не скомпилируется
3. **`.gitignore`** — пустой файл, node_modules/.env попадут в коммит
4. **`.env.example`** — нет шаблона переменных окружения (OpenRouter API key, Firestore config)

### Важно (нужно вскоре после старта)

5. **Схемы API запросов/ответов** — эндпоинты перечислены, но нет JSON-схем (request/response bodies)
6. **Схема данных Firestore** — `sessions/{id}` и `users_state/{id}` упомянуты, но структура не определена
7. **Протокол общения агентов** — формат сообщений между агентами не задан
8. **Решение по хранилищу** — "Firestore или Redis" — не выбрано окончательно
9. **Решение по тест-фреймворку** — Vitest или Jest — не выбрано

### Мелочи

10. **Путь в `AGENTS.md`** — `ai-docs/tech/restrictions/restrictions.md` ≠ реальный путь `ai-docs/restrictions/restrictions.md`
11. **`restrictions.md`** — описано только 1 правило, нет деталей по prompt injection, input sanitization, tool access control
12. **`README.MD`** — placeholder "Let's start!" — нужен нормальный README
13. **Нет UI мокапов** — интерфейс чата не описан

### Нерешённые вопросы

- **LLM интеграция**: LiteLLM proxy (tech/main.md) vs OpenRouter напрямую (AGENTS.md) — какое решение?
- **Kaggle трек**: Freestyle или Agents for Business?
- **Dockerfile/cloudbuild.yaml** — упомянуты в плане, но не созданы

- **Хранилище:** Redis

- **Тесты:** Vitest

2. Закрыть мелкие вопросы (UI мокапы, схемы данных, протокол агентов)
3. Начать с backend слоя (Fastify + ADK агенты)
