# AI Job Interview Simulator — Полный план

## Контекст

Проект Kaggle Capstone. Зелёный поле — нет ни одного файла кода. Нужно построить полноценный AI Interview Simulator с 5 агентами, задеплоить на Google Cloud Run.

**Дедлайн:** 6 июля 2026
**Стек:** Google ADK (TS) + DeepSeek via OpenRouter + Fastify + Next.js + Redis + Cloud Run

---

## Решения по архитектуре

### LLM: DeepSeek через OpenRouter
- TypeScript ADK не имеет встроенного LiteLLM-коннектора (как Python)
- Решение: используем **Google GenAI SDK** с кастомным `baseUrl` pointing to OpenRouter
- OpenRouter совместим с OpenAI API форматом
- Модель: `deepseek/deepseek-chat` через OpenRouter

### ADK: TypeScript (`@google/adk`)
- Официальный пакет `@google/adk`
- `LlmAgent` + `FunctionTool` для каждого из 5 агентов
- `SequentialWorkflowAgent` для orchestration pipeline

### Хранилище: Redis (ioredis)
- Проще для dev + Cloud Run (stateless)

### Шаг 1.1: LLM клиент
**Файл:** `src/config.ts`

Создать обёртку над OpenRouter API. OpenRouter использует OpenAI-совместимый API:
- Base URL: `https://openrouter.ai/api/v1`
- Headers: `Authorization: Bearer $OPENROUTER_API_KEY`
- Модель: `deepseek/deepseek-chat`

### Шаг 1.2: Минимальный ADK агент
**Файл:** `src/agents/root.ts`

```typescript
import { LlmAgent } from '@google/adk';

export const rootAgent = new LlmAgent({
  name: 'interview_bot',
  model: 'gemini-flash-latest', // temporarily use Gemini for dev
  instruction: 'You are a helpful interview assistant.',
});
```

> **Важно:** TypeScript ADK по умолчанию работает с Gemini. Для DeepSeek нужно:
> 1. Либо использовать `GEMINI_API_KEY` для dev/тестирования
> 2. Либо настроить прокси OpenRouter → ADK через httpOptions
> 3. Либо написать кастомный ModelAdapter

**Рекомендация для Phase 1:** Использовать бесплатный Gemini API для быстрого прототипа, DeepSeek подключить в Phase 2.

## Phase 2: 5 ADK Агентов (День 3-5)

### Шаг 2.1: Определение типов
**Файл:** `src/types/index.ts`

```typescript
interface JobProfile {
  role: string;
  level: 'junior' | 'middle' | 'senior';
  skills: string[];
  domain: string;
  keywords: string[];
}

interface Question {
  id: string;
  text: string;
  topic: string;
  difficulty: number;
}

interface Evaluation {
  score: number;
  strengths: string[];
  weaknesses: string[];
  recommendation: string;
}

interface Session {
  id: string;
  jobProfile: JobProfile;
  history: Array<{ question: string; answer: string; evaluation: Evaluation }>;
  weakTopics: string[];
  currentDifficulty: number;
}
```

### Шаг 2.2: FunctionTools
**Файлы:** `src/tools/*.ts`

Каждый tool — `FunctionTool` из `@google/adk`:

1. **`parseJob.ts`** — парсит текст вакансии → `JobProfile`
   - LLM вызов с structured output (JSON schema)
   - Записывает результат в Redis

2. **`generateQuestion.ts`** — генерирует вопрос по `JobProfile` + `weakTopics`
   - Учитывает текущую сложность
   - Фокусируется на слабых темах

3. **`evaluateAnswer.ts`** — оценивает ответ → `Evaluation`
   - Структурированный вывод: score, strengths, weaknesses

4. **`updateMemory.ts`** — обновляет `weakTopics` в сессии
   - Добавляет/убирает слабые темы

5. **`fetchWeakTopics.ts`** — получает текущие слабые темы

### Шаг 2.3: Агенты
**Файлы:** `src/agents/*.ts`

1. **JobParserAgent** — использует `parseJob` tool
2. **InterviewerAgent** — использует `generateQuestion` tool
3. **EvaluatorAgent** — использует `evaluateAnswer` tool
4. **CoachAgent** — объясняет правильный ответ, даёт советы
5. **MemoryAgent** — использует `updateMemory` + `fetchWeakTopics`

### Шаг 2.4: Оркестрация (Sequential Workflow)
**Файл:** `src/agents/root.ts`

```
Pipeline:
1. User вводит вакансию → JobParser → создаёт JobProfile
2. Interviewer → генерирует вопрос
3. User отвечает → Evaluator → оценивает
4. Coach → объясняет
5. Memory → обновляет слабые темы
6. Interviewer → следующий вопрос (loop)
```

Используем `SequentialWorkflowAgent` или ручной orchestration через Fastify routes.

### Шаг 2.5: API Routes
**Файлы:** `src/routes/*.ts`

- `POST /job/parse` — парсит вакансию, создаёт сессию
- `POST /interview/start` — начинает интервью
- `POST /interview/answer` — принимает ответ, возвращает следующий вопрос + фидбек
- `GET /session/:id` — получает данные сессии

### Шаг 2.6: Redis сервис
**Файл:** `src/services/redis.ts`

```typescript
// CRUD для сессий
createSession(session: Session): Promise<void>
getSession(id: string): Promise<Session | null>
updateSession(id: string, data: Partial<Session>): Promise<void>
```
