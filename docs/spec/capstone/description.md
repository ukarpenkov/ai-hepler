# AI Agents: Intensive Vibe Coding Capstone Project

> Put your course learning into action by building an agent that solves a real-world problem, helps people or improves everyday living.

**Deadline:** Jul 7, 2026 at 9:59 AM GMT+3

---

## Overview

The AI Agents: Intensive Vibe Coding Capstone Project invites participants to build an AI agent using the concepts, tools, and best practices learned in Kaggle's 5-Day AI Agents: Intensive Vibe Coding Course with Google. Participants will create a practical, real-world solution that demonstrates how AI agents can automate tasks, assist users, solve business challenges, or improve everyday workflows.

Submissions should showcase both the project's value and technical implementation through a Kaggle Writeup, public codebase, video demonstration, and project link. Projects may be entered into one of four tracks and will be evaluated based on innovation, solution design, communication, and the effective application of course concepts and agent technologies.

---

## Submission Requirements

A valid submission **must** contain:

1. **Kaggle Writeup**
2. **Media Gallery**
3. **Attached Public Video**
4. **Attached Project Link**

> Your final Submission must be made prior to the deadline. Any un-submitted or draft Writeups by the hackathon deadline will not be considered by the Judges.

### 1. Kaggle Writeup

The Kaggle Writeup serves as your project report. It should include a title, subtitle, and a detailed analysis of your submission. You must select a Track for your Writeup in order to submit.

- **Max length:** 2,500 words. Submissions over this limit may be subject to penalty.

### 1a. Media Gallery

Attach any images and/or videos associated with your submission. A cover image is required to submit your Writeup.

### 1b. Video

Videos should be **5 minutes or less**, and should be published to YouTube.

### 1c. Public Project Link

A URL to your working product or interactive demo. Must be publicly accessible and not require a login or paywall. If a live demo is not feasible, a link to your public code repository (e.g., GitHub) is required, including detailed setup instructions.

> If you attach a private Kaggle Resource to your public Kaggle Writeup, your private Resource will automatically be made public after the deadline.

## Evaluation Criteria

You must demonstrate at least **three (3)** key concepts from the course:

| Key Concept | Where to Demonstrate |
|---|---|
| Agent / Multi-agent system (ADK) | Code |
| MCP Server | Code |
| Antigravity | Video |
| Security features | Code or Video |
| Deployability | Video |
| Agent skills (e.g., Agents CLI) | Code or Video |

### Category 1: The Pitch — Problem, Solution, Value (30 points)

| Criteria | Points | Description |
|---|---|---|
| Core Concept & Value | 10 | Central idea, relevance to track, innovation, value. Agents should be clear, meaningful and central to the solution. |
| YouTube Video | 10 | Clarity, conciseness, quality of messaging. Should cover: Problem Statement, Why agents, Architecture, Demo, The Build. |
| Writeup | 10 | How well the written submission articulates problem, solution, architecture, and project journey. |

**Video should cover (within 5 min limit):**

- **Problem Statement:** Describe the problem and why it matters.
- **Agents:** Why agents? How can agents uniquely help solve that problem?
- **Architecture:** Images and description of the overall agent architecture.
- **Demo:** Demo of the solution (images, animation, or video of the agent working).
- **The Build:** How you created it, tools and technologies used.

### Category 2: The Implementation — Architecture, Code (70 points)

| Criteria | Points | Description |
|---|---|---|
| Technical Implementation | 50 | Quality of architecture, code, meaningful use of agents, tool use (especially clever usage of existing toolsets). Code should contain comments pertinent to implementation, design and behaviors. |
| Documentation | 20 | README.md with: problem description, solution, architecture, setup instructions, and relevant diagrams/images. |

> Participants are not required to deploy agents to a live public endpoint. If you do deploy, provide documentation to reproduce the deployment.

> **DO NOT INCLUDE ANY API KEYS OR PASSWORDS IN YOUR CODE.**
