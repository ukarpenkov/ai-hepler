# Phase 5 — Step 9: Agent Workflow Integration Test (with Mocked LLM)

**Status:** Completed

## What Was Done

### `src/__tests__/agentWorkflow.test.ts`

Created an integration test for the full agent workflow:

1. **Full interview cycle:**
   - `jobParserAgent` → `ParsedJob` (role, skills, domain)
   - `interviewerAgent` → `Question` (question, topic, difficulty)
   - `evaluatorAgent` → `Evaluation` (score, strengths, weaknesses)
   - `memoryAgent` → `MemoryUpdate` (weakSkills, answeredTopics)
   - `fetchWeakTopicsTool` → `string[]`

2. **Call order:** Each step is called sequentially, data is passed correctly between steps

3. **Error handling:** LLM API error (500) propagates through `jobParserAgent` with message "LLM API error: 500"

4. **Tool Access Control:** 3 tests verify:
   - System tools are blocked for external context
   - All tools are accessible from agent context
   - Memory tools are accessible from route context, system tools are not

5. **Mocks:**
   - LLM client replaced via `vi.stubGlobal("fetch", ...)` with predictable responses
   - Redis replaced via `vi.mock("ioredis", ...)` with in-memory store
   - Tool calls go through agents (not directly)
