# Bug: Evaluation card appears after first question, blocks chat

**Дата:** 2026-06-21
**Приоритет:** High
**Статус:** Open
**Компонент:** Frontend — `ChatWindow.tsx`, `FeedbackCard.tsx`

## Описание

После отправки первого ответа на интервью, полная карточка оценки (FeedbackCard) появляется сразу внизу экрана. Это занимает слишком много места и делает окно чата узким и нечитаемым.

## Ожидаемое поведение

- Вопрос 1-9: после ответа показывать только **краткую обратную связь** (балл + 1 совет) внутри чата, без полной карточки
- Вопрос 10 (последний): показать **полную итоговую оценку** со всеми 10 ответами, статистикой и рекомендациями

## Фактическое поведение

- После **первого же ответа** появляется полная карточка FeedbackCard с оценкой, сильными/слабыми сторонами, рекомендацией и разбором ответа
- Карточка отображается **вне области прокрутки** чата, в блоке с input
- Это сжимает окно ввода и делает интерфейс нечитаемым
- Пользователь видит "итоговую оценку" уже на вопросе 2 из 10

## Воспроизведение

1. Открыть http://localhost:3000
2. Вставить описание вакансии, начать интервью
3. Ответить на первый вопрос
4. **Наблюдение:** сразу появляется полная карточка "Оценка 6/10" с разбором

## Где исправлять

**`packages/web/components/ChatWindow.tsx`:**
- FeedbackCard рендерится при `lastFeedback && lastCoach` (строка ~112)
- Не проверяется `questionCount` — показывается всегда после первого ответа

**Исправление:**
- Показывать FeedbackCard **только когда** `questionCount >= TOTAL_QUESTIONS` (10)
- До этого — показывать inline-уведомление "Оценка: X/10. Совет: ..." внутри чата
- После 10 вопросов — показать финальную сводку со всеми оценками и статистикой

## Компоненты для изменения

| Файл | Изменение |
|------|-----------|
| `packages/web/components/ChatWindow.tsx` | Добавить условие `isFinished` для показа FeedbackCard, собирать все ответы для итоговой сводки |
| `packages/web/components/FeedbackCard.tsx` | Опционально: поддержка compact-режима для inline показа |
