# Fix: jobProfile is required

## Root Cause

Frontend API wrapper `packages/web/lib/api.ts` не передаёт `jobProfile` и `sessionData` на бэкенд:

- `startInterview(sessionId)` — принимает только `sessionId`, caller передаёт `{jobProfile, weakSkills, history}` вторым аргументом, но функция его игнорирует
- `sendAnswer(sessionId, answer)` — принимает только 2 аргумента, caller передаёт `{jobProfile, weakSkills, history}` третьим, но функция его игнорирует

Бэкенд (`src/api/routes/interview.ts:22,63`) требует `jobProfile` и возвращает 400.

## Changes

### `packages/web/lib/api.ts`

1. `startInterview` — добавить второй параметр `sessionData: { jobProfile, weakSkills, history }`, передать в `JSON.stringify`
2. `sendAnswer` — добавить третий параметр `sessionData: { jobProfile, weakSkills, history }`, передать в `JSON.stringify` как `sessionData`
