# Fix: jobProfile is required

## Root Cause

Frontend API wrapper `packages/web/lib/api.ts` doesn't send `jobProfile` and `sessionData` to the backend:

- `startInterview(sessionId)` — only accepts `sessionId`, caller passes `{jobProfile, weakSkills, history}` as second argument but the function ignores it
- `sendAnswer(sessionId, answer)` — only accepts 2 arguments, caller passes `{jobProfile, weakSkills, history}` as third but the function ignores it

Backend (`src/api/routes/interview.ts:22,63`) requires `jobProfile` and returns 400.

## Changes

### `packages/web/lib/api.ts`

1. `startInterview` — add second parameter `sessionData: { jobProfile, weakSkills, history }`, pass to `JSON.stringify`
2. `sendAnswer` — add third parameter `sessionData: { jobProfile, weakSkills, history }`, pass to `JSON.stringify` as `sessionData`
