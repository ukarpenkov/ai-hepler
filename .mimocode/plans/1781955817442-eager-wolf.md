## What's missing to start

### Critical (needed BEFORE writing code)

1. **`package.json`** — doesn't exist at all, impossible to start
2. **`tsconfig.json`** — TypeScript won't compile
3. **`.gitignore`** — empty file, node_modules/.env will end up in commits
4. **`.env.example`** — no environment variable template (OpenRouter API key, Firestore config)

### Important (needed shortly after starting)

5. **API request/response schemas** — endpoints listed, but no JSON schemas (request/response bodies)
6. **Firestore data schema** — `sessions/{id}` and `users_state/{id}` mentioned, but structure undefined
7. **Agent communication protocol** — message format between agents not defined
8. **Storage decision** — "Firestore or Redis" — not finalized
9. **Test framework decision** — Vitest or Jest — not chosen

### Minor issues

10. **Path in `AGENTS.md`** — `ai-docs/tech/restrictions/restrictions.md` ≠ actual path `ai-docs/restrictions/restrictions.md`
11. **`restrictions.md`** — only 1 rule described, no details on prompt injection, input sanitization, tool access control
12. **`README.MD`** — placeholder "Let's start!" — needs a proper README
13. **No UI mockups** — chat interface not described

### Open questions

- **LLM integration**: LiteLLM proxy (tech/main.md) vs OpenRouter directly (AGENTS.md) — which decision?
- **Kaggle track**: Freestyle or Agents for Business?
- **Dockerfile/cloudbuild.yaml** — mentioned in the plan but not created

## Decisions Made

- **Storage:** Redis
- **LLM:** DeepSeek via Google GenAI SDK with baseUrl replacement (Option 1 from tech/main.md)
- **Tests:** Vitest
- **Kaggle track:** Agents for Business

## Recommendation

Documentation covers ~80% of planning. To start you need:
1. Create `package.json`, `tsconfig.json`, `.gitignore`, `.env.example`
2. Resolve minor issues (UI mockups, data schemas, agent protocol)
3. Start with the backend layer (Fastify + ADK agents)
