### `src/__tests__/agentWorkflow.test.ts`

Создан интеграционный тест полного workflow агентов:

1. **Полный цикл интервью:**
   - `jobParserAgent` → `ParsedJob` (role, skills, domain)
   - `interviewerAgent` → `Question` (question, topic, difficulty)
   - `evaluatorAgent` → `Evaluation` (score, strengths, weaknesses)
   - `memoryAgent` → `MemoryUpdate` (weakSkills, answeredTopics)
   - `fetchWeakTopicsTool` → `string[]`

2. **Порядок вызовов:** Каждый step вызывается последовательно, данные передаются между шагами корректно

3. **Обработка ошибок:** LLM API error (500) пробрасывается через `jobParserAgent` с сообщением "LLM API error: 500"

4. **Tool Access Control:** 3 теста проверяют:
   - System tools заблокированы для external контекста
   - Все tools доступны из agent контекста
   - Memory tools доступны из route контекста, system tools — нет

5. **Моки:**
   - LLM клиент заменён через `vi.stubGlobal("fetch", ...)` с предсказуемыми ответами
   - Redis заменён через `vi.mock("ioredis", ...)` с in-memory store
